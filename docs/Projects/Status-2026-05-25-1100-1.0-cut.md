## Non-obvious state

- **`FeedItem.reply` / `.reason` are `serde_json::Value`, not typed structs**. A strict-struct attempt got a 1.0 cut empty-feed bug for accounts with V1 prefs / odd embeds. The defensive shape stays. If anyone "improves" it to typed structs, watch for empty-feed regressions.
- **`use_resource` reactivity gotcha**. Several sheets (Profile, Thread, SavedFeeds) hit a "captured key by value → fetch frozen" bug at 1.0. The pattern is: read the focus signal **inside** the async closure, not at the outer render. Documented in [[../Architecture/Architecture-Overview#Reactive use_resource gotcha]].
- **`safe_open` allowlist** is load-bearing security. Don't add a `Command::new("open").arg(url)` site without routing through `crate::safe_open::open_in_browser`. ADR-002 explains why.
- **Session file > Keychain** (ADR-001). Don't reach for `keyring` even though it looks tidier.
- **`publish = false` is deliberate** (ADR-003). Don't flip it during a release-plz config tweak.
